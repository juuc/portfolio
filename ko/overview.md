## 요약

2025년 3월 부톡(Bootalk)에 **CDO(Chief Data Officer)**로 합류하여, 데이터를 넘어 엔지니어링 조직 전체로 역할을 빠르게 확장. 2025년 8월 CTO 퇴사 이후 **Tech Lead / Product Owner** 역할을 수행 — 4인 개발팀 관리, 전체 플랫폼 배포 책임, 웹/모바일/백엔드/데이터 전반의 제품 의사결정을 주도.

11개월간 **28개 레포지토리**에 **3,203건의 커밋**과 **754건 이상의 풀 리퀘스트**를 기여 — 빌더이자 리더로서 활동한 비정상적으로 넓은 범위를 반영.

## AX: AI 전환 — 핵심 역량으로서의 AI 활용

AI 활용은 선택이 아닌 **핵심 엔지니어링 역량** — 단일 개발자가 달성할 수 있는 것을 근본적으로 바꿉니다.

### Cline → Claude Code 전환

2025년 8월 이전에는 **Cline** (VS Code 확장)을 AI 코딩 보조 도구로 사용했습니다. 유용했지만 한계가 있었습니다 — 단일 파일 컨텍스트, 자율 실행 불가, 병렬 처리 불가.

**2025년 8월 16일**, **Claude Code** (CLI)를 도입했습니다. 첫 `CLAUDE.md`가 오후 1시 14분에 커밋되었고, 첫 Claude 보조 코드 커밋 — Expo 마이그레이션 1단계 — 이 11분 후에 생성되었습니다. 10일 이내에 모든 주요 레포지토리에 Claude Code 워크플로우를 배포했습니다:

| 날짜 | 레포 | 이벤트 |
|------|------|--------|
| 8월 16일 | bootalk_app | 첫 `CLAUDE.md` + 첫 Claude 보조 코드 |
| 8월 19일 | legacy-service | Claude PR 리뷰 워크플로우 |
| 8월 25일 | btalk2.1_backend | Claude PR 리뷰 워크플로우 |
| 9월 05일 | data | Claude PR 리뷰 워크플로우 |
| 9월 08일 | ubuntu-crawler | 첫 Claude 공동 저작 커밋 |
| 9월 22일 | bootalk-amplify | Claude PR 리뷰 워크플로우 |
| 10월 29일 | frontend_onboarding | 첫 Claude 공동 저작 커밋 |

이후 팀 전체 개발자를 위해 **Max 20x 플랜**을 구매했습니다. 이것은 단순한 개인 생산성 도구가 아닌 — 팀 전체의 소프트웨어 배포 방식을 바꾼 **조직 차원의 AX(AI Transformation)** 결정이었습니다.

### 숫자가 말하는 이야기

| 기간 | 월간 커밋 | AI 도구 |
|------|----------|---------|
| 3~7월 | 평균 ~65/월 | Cline (VS Code) |
| **8월** | **291** | **Claude Code 도입** |
| 9월 | 163 | 워크플로우 개선 |
| 10월 | 396 | 멀티 레포 배포 완료 |
| 11월 | 632 | 병렬 오케스트레이션 숙달 |
| 12월 | **838** | 최고치 — Claude 도입 전 대비 13배 |

Cline 사용 시 월 ~65건에서 Claude Code로 **12월 838건** — **13배 증가**. 이것은 빠르게 타이핑하는 것이 아닙니다. 근본적으로 다른 작업 방식입니다.

### 워크플로우 동작 방식

**Tmux 멀티 페인 오케스트레이션:**
- 4-8개 Claude Code 에이전트를 분할 터미널 패널에서 동시 실행
- 각 에이전트가 비중복 파일셋을 소유하여 충돌 방지
- 에이전트가 자율 작업 — 코드베이스 읽기, 변경 계획, 구현, 자체 검증

**야간 자율 실행:**
- 복잡한 작업을 큐에 추가 (타입 안전성 페이즈, 테스트 스위트, 리팩토링 캠페인)
- 에이전트가 밤새 연속 처리
- 아침에 리뷰 가능한 완성된 PR 확인

**페이즈 기반 병렬 처리:**
- 대규모 캠페인을 독립적 페이즈로 분해 (예: 50+ PR 타입 안전성 에픽)
- 동시 에이전트 세션에 페이즈 분산
- 공유 `CLAUDE.md` 지침과 이슈 트래킹으로 조율

이 방법론을 통해 단일 엔지니어가 5인 이상 팀에 필적하는 산출물 생산 — 11개월간 28개 레포지토리에 걸친 3,203건의 커밋과 754건 이상의 PR을 가능하게 한 직접적 원동력.

## 구축한 것들

웹(Next.js), 모바일(React Native/Expo), 어드민 시스템 전반의 프론트엔드 아키텍처를 리드하며 동시에 데이터 파이프라인, 백엔드 서비스, AI 프로덕트를 구축. 클라우드 마이그레이션, 타입 안전성 캠페인, 포괄적인 테스트 커버리지 등 주요 플랫폼 현대화 이니셔티브를 주도.
