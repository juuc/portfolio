# 자율 Sentry 트리아지 봇

[← 포털로 돌아가기](../../README.md) | [English](../../en/projects/sentry-triage-bot.md)

## 프로젝트 개요

Sentry 웹훅을 수신하여 4개 레포지토리에 걸쳐 프로덕션 이슈를 진단하고, 수정 PR을 자동 생성하는 자율 에러 트리아지 파이프라인. OpenClaw(AI 에이전트 오케스트레이션 프레임워크)와 Cloudflare Tunnel 기반 보안 웹훅 수신으로 구축.

## 문제

수동 Sentry 트리아지가 엔지니어링 집중력을 지속적으로 소모:

- **컨텍스트 스위칭:** 모든 알림마다 현재 작업을 중단하고, 에러를 읽고, 어느 레포/환경에서 발생했는지 확인하고, 대응 방법을 결정해야 함
- **크로스 레포 진단:** 이슈가 프론트엔드, 백엔드, 레거시 서비스, 서버리스 레이어에 걸쳐 있는 경우가 빈번 — 각각 별도 조사 필요
- **환경 혼동:** 개발/프로덕션 알림이 뒤섞여 비프로덕션 노이즈 트리아지에 시간 낭비
- **온콜 피로:** CTO 퇴사 후 유일한 Tech Lead로서 모든 Sentry 알림이 궁극적으로 본인에게 도달

## 해결

### 아키텍처

```
Sentry 웹훅
  → Cloudflare Tunnel (공개 엔드포인트)
    → 토큰 프록시 (인증 주입)
      → OpenClaw 게이트웨이 (웹훅 핸들러)
        → Transform 레이어 (노이즈 필터링 + 트리아지 프롬프트 구성)
          → AI 에이전트 (자율 진단)
            → 텔레그램 그룹 (트리아지 리포트)
            → GitHub PR (프로덕션 수정)
            → Sentry API (이슈 해결)
```

### 동작 방식

**1. 지능형 노이즈 필터링**

Transform 레이어가 AI 에이전트 실행 전에 수신 웹훅을 분류 — 테스트 핑, 해결된 이벤트, 불완전한 페이로드는 토큰 소비 없이 처리. 실제 조치 가능한 이슈만 AI에 전달.

**2. 필수 증거 수집**

에이전트는 웹훅 페이로드만으로 절대 분류하지 않음. 모든 이슈에 대해:
- Sentry API에서 전체 이슈 상세 및 최신 이벤트 조회
- 호스트, 릴리스 태그, 브레드크럼, URL에서 환경 증거 추출
- 프로덕션/개발 환경 여부 판별

**3. 크로스 레포지토리 검증**

모든 이슈를 4개 코드베이스에서 조사:
- 프론트엔드 모노레포 (Next.js/React)
- Kotlin/Spring 백엔드
- 레거시 Node.js 서비스
- AWS Lambda (Amplify) 레이어

필요 시 읽기 전용 DB 쿼리(MySQL, pymysql)나 AWS CLI 조회로 근본 원인 검증 — 모든 쓰기 작업은 하드 금지.

**4. 환경 인식 정책 게이트**

| 환경 | 수정 PR | GitHub Issue | Sentry 조치 |
|------|--------|-------------|------------|
| 프로덕션 | 워크트리에서 생성, 통합 브랜치 타겟 | 허용 | Resolve (inNextRelease) |
| 개발 | 금지 | 금지 | 증거 기반 resolve/keep open |

개발 환경의 노이즈 PR을 방지하면서 프로덕션 이슈에는 자동 수정을 보장.

**5. 안전 제어**

- 모든 코드 수정은 `/tmp` 워크트리에서 수행 — 레포 루트에서 절대 직접 수정 안 함
- PR은 통합 브랜치 타겟 (`main` 직접 타겟 금지)
- 읽기 전용 DB 접근만 허용 — 쓰기/삭제 작업 하드 차단
- 패치 실패 복구: 재읽기 → 헝크 분리 → 3회 재시도 → 차단된 헝크 보고
- 모든 조사 이슈를 `incidents-YYYY-MM.jsonl` 및 `pr-history.md`에 기록

**6. 헬스 모니터링**

15분 간격 크론 잡이 전체 파이프라인 점검:
- Cloudflare Tunnel 프로세스 생존 확인
- 로컬 웹훅 엔드포인트 응답 (202)
- 공개 엔드포인트 응답 (202)

정상 시 무음; 장애 시에만 텔레그램 알림.

## 비즈니스 임팩트

| 지표 | 이전 | 이후 |
|------|------|------|
| 트리아지 응답 시간 | 수분~수시간 (사람) | 수초 (자율) |
| 크로스 레포 조사 | 수동, 4개 별도 체크아웃 | 자동, 4개 레포 병렬 |
| 개발 환경 노이즈 | 프로덕션 알림과 혼재 | 자동 필터링 |
| 수정 배포 | 수동: 진단 → 코딩 → PR → 리뷰 | 자동: 웹훅 → 수분 내 PR |
| 온콜 부담 | 모든 알림에 사람 필요 | 에지 케이스만 사람 리뷰 |

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 웹훅 수신 | Cloudflare Tunnel + Node.js 프록시 |
| 에이전트 오케스트레이션 | OpenClaw (Gemini 3.1 Pro) |
| 트리아지 로직 | JavaScript transform + 마크다운 런북 |
| 코드 분석 | 멀티 레포 워크트리 격리 |
| DB 검증 | pymysql (읽기 전용) + AWS CLI |
| 알림 | Telegram Bot API |
| 헬스 모니터링 | 크론 기반 엔드포인트 폴링 |
| 버전 관리 | Git 워크트리 우선 안전 패턴 |

## 이 프로젝트가 중요한 이유

이 프로젝트는 "AI가 코드 작성을 도와준다"에서 "AI가 프로덕션 시스템을 자율 운영한다"로의 진화를 보여줍니다. 챗봇이나 코드 생성기가 아닌 — 잠들지 않고, 전체 코드베이스 컨텍스트로 모든 알림을 조사하고, 사람은 리뷰하고 머지만 하면 되는 수정 PR을 생산하는 온콜 엔지니어입니다.

5가지 AX 원칙(인프라 마인드셋, 의도 위임, 조율 설계, 품질 소유, 팀 확장)이 하나의 시스템에 수렴 — AI 기반 엔지니어링이 빠른 타이핑 이상임을 증명하는 실질적 근거입니다.
