## 비즈니스 임팩트

### 1. 실거래가 데이터 파이프라인

**문제:** 부톡에 정부 출처의 정확하고 최신인 부동산 거래 데이터 필요.

**해결:** 공공 API에서 실거래가 데이터를 자동으로 수집하는 파이프라인 구축. 일별 통계 산출, 아파트 ID 매핑, 변동 감지 시 Telegram 알림.

**임팩트:** 부톡의 부동산 가치 평가와 시장 인사이트를 구동하는 핵심 데이터 자산.

### 2. Dagster 마이그레이션 — Cron에서 현대적 오케스트레이션으로

**문제:** 크롤러가 subprocess 호출 기반 cron job으로 동작 — 모니터링, 재시도, 의존성 관리 없음. 실패가 눈에 보이지 않음.

**해결:** 모든 크롤러를 Dagster 네이티브 에셋으로 마이그레이션: 아파트 랭킹, 대출 상품, 실거래가. subprocess 호출을 네이티브 Python으로 교체. 비동기 지원 추가.

**임팩트:** Dagster UI를 통한 완전한 관측성. 자동 재시도. 의존성 인식 스케줄링.

### 3. 아파트 랭킹 시스템

**문제:** 사용자가 다양한 기준으로 아파트를 비교하길 원함.

**해결:** 서울, 경기, 인천 지역 아파트 랭킹을 위한 엔드투엔드 데이터 ETL 구축.

**임팩트:** 부톡 웹/앱의 랭킹 기능 구동.

### 4. 분양 & 재건축 데이터

**문제:** 신축 아파트를 찾는 사용자에게 분양 정보 필요. 재건축 데이터도 DB 제약 조건으로 불완전.

**해결:** 주간 스케줄링(매주 월요일 19:00 KST) 기반 분양 데이터 수집. 트랜잭션 안전을 위해 TRUNCATE 대신 DELETE 사용. 2026년 1월 기준 분양 데이터 **매주 완벽 최신화 완료**. 매물 유형 필터 수정으로 재건축 단지도 수집하도록 개선. 저장 프로시저의 필터링 제약으로 인한 재건축 데이터 가시성 이슈를 발견하고 문서화.

**임팩트:** 부톡 사용자가 시기적절한 분양 및 재건축 정보로 의사결정 가능. 매주 완전한 분양 데이터 업데이트.

### 5. 대출 크롤러

**문제:** 부동산 데이터와 함께 대출 정보 필요.

**해결:** 전세자금대출, 주택담보대출 상품 크롤러 구축.

**임팩트:** 매물 정보와 함께 포괄적 금융 정보 제공.

### 6. 코드 아키텍처 — Repository 패턴 & Import 표준

**해결:** Import 규칙 표준화 (도메인 내부: 상대 경로, 공유: 절대 경로). Repository 패턴 사용 가이드 문서화.

**임팩트:** 유지보수 가능하고 일관된 코드베이스. 신규 개발자 온보딩 용이.
